# Auto Quote Maker — Aerial Image Dataset Builder

A two-script pipeline that discovers residential addresses in a neighborhood and downloads satellite images of each property at a consistent resolution and zoom level. Images are intended for manual annotation in **CVAT** to classify features like driveways, patios, and walkways.

---

## How it works

```
populate_addresses.py          fetch_aerial_views.py
─────────────────────          ─────────────────────
Reference address              addresses.csv
      │                               │
      ▼                               ▼
Google Geocoding API       Google Maps Static API
      │                               │
      ▼                               ▼
Overpass API (OSM)             data/*.png (640x640)
      │
      ▼
addresses.csv
```

**Step 1** — `populate_addresses.py` takes a single reference address, finds its coordinates via the Google Geocoding API, then queries OpenStreetMap's Overpass API to enumerate all residential addresses within a given radius. Results are written to `addresses.csv`.

**Step 2** — `fetch_aerial_views.py` reads `addresses.csv` and downloads a satellite image for each property using the Google Maps Static API. Every image is identical in zoom, resolution, and format — a hard requirement for fair comparison during labeling.

---

## Requirements

- Python 3.10+
- No third-party packages — only the standard library
- A Google Cloud project with two APIs enabled:
  - **Geocoding API**
  - **Maps Static API**

---

## Setup

### 1. Clone / enter the project directory

```bash
cd auto-quote-maker
```

### 2. Add your Google Maps API key

Create a `.env` file in the project root:

```
GOOGLE_MAPS_API_KEY=your_key_here
```

Both scripts read this file automatically. The key is never hard-coded.

> To get a key: Google Cloud Console → APIs & Services → Credentials → Create API Key.
> Then enable "Geocoding API" and "Maps Static API" for that key.

### 3. Verify the directory structure

```
auto-quote-maker/
├── .env                    ← your API key goes here
├── addresses.csv           ← generated by populate_addresses.py
├── populate_addresses.py
├── fetch_aerial_views.py
└── data/                   ← satellite images land here
```

## Usage

### Step 1 — Discover addresses in your neighborhood

```bash
python3 populate_addresses.py --reference "9212 Cedarcrest Dr, Bethesda, MD" --limit 150 --radius 1000
```

| Flag | Required | Default | Description |
|------|----------|---------|-------------|
| `--reference` | yes | — | Any address in the target neighborhood. Used as the center point for the search. |
| `--limit` | no | `100` | Maximum number of addresses to write to the CSV. |
| `--radius` | no | `400` | Search radius in meters around the reference address. 400m ≈ 2–3 blocks; 1000m covers a larger neighborhood. |
| `--out` | no | `addresses.csv` | Output CSV file path. |

**Output** — `addresses.csv` with columns: `address`, `lat`, `lng`

```
address,lat,lng
9203 Cypress Avenue,39.006007,-77.102398
9205 Cypress Avenue,39.005891,-77.102461
...
```

> **Note on coverage:** Address discovery depends on OpenStreetMap data. Newer or rural neighborhoods may have fewer tagged addresses. If you get fewer results than expected, try increasing `--radius`. If OSM coverage is sparse in your area, you can also manually add rows to `addresses.csv`.

---

### Step 2 — Download satellite images

```bash
python3 fetch_aerial_views.py
```

| Flag | Required | Default | Description |
|------|----------|---------|-------------|
| `--input` | no | `addresses.csv` | CSV file to read addresses from. |
| `--out-dir` | no | `data/` | Directory where images are saved. |

**Output** — one PNG per address in `data/`:

```
data/
├── 9203_Cypress_Avenue.png
├── 9205_Cypress_Avenue.png
└── ...
```

Each image is:
- **640 × 640 pixels**
- **Zoom level 20** (~50m across — shows the full property clearly)
- **Satellite** map type (aerial view, no labels)
- **PNG** format

> Re-running `fetch_aerial_views.py` is safe — already-downloaded images are skipped automatically.

## Image specifications

These values are locked in `fetch_aerial_views.py` and must not be changed between runs. Changing them mid-dataset would make images incomparable in CVAT.

| Property | Value |
|----------|-------|
| Resolution | 640 × 640 px |
| Zoom | 20 |
| Scale | 1× |
| Map type | satellite |
| Format | PNG |

---

## Importing into CVAT

1. Create a new CVAT project and task.
2. Upload the contents of `data/` as the image set.
3. Define your label set — suggested labels for this dataset:

   | Label | Description |
   |-------|-------------|
   | `driveway` | Paved or gravel vehicle access from street to garage/carport |
   | `patio` | Hard-surface outdoor area attached to the house |
   | `walkway` | Foot path from street/driveway to front door or around the property |

4. Use polygon or bounding-box annotations depending on your downstream model requirements.

## Troubleshooting

**`GOOGLE_MAPS_API_KEY not found`**
Ensure `.env` exists in the same directory you run the scripts from and contains `GOOGLE_MAPS_API_KEY=...` with no extra spaces.

**`Geocoding failed: REQUEST_DENIED`**
The Geocoding API is not enabled for your key, or billing is not set up on your Google Cloud project.

**`HTTP Error 403` when downloading images**
The Maps Static API is not enabled for your key.

**Overpass mirror warnings (`[warn] ... failed`)**
Normal — the script automatically tries three Overpass mirrors. As long as one succeeds, output is unaffected.

**Fewer addresses than expected**
OSM coverage is incomplete in your area. Increase `--radius`, or manually add rows to `addresses.csv` in the format `address,lat,lng`.

**Images look identical / wrong location**
Verify the geocoded coordinates printed during Step 1 are correct before running Step 2. If the reference address is ambiguous, make it more specific (add city, state, zip).
